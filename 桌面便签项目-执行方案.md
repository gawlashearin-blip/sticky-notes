# 桌面便签程序 — 执行方案（WPF + C#）

> 把本文件放到项目根目录（例如 `D:\projects\sticky-notes\PLAN.md`），方便 Claude Code 读取整体规划。

## 最终方案锁定

**技术栈：** WPF + C# (.NET 8) + VS Code
**核心设计：** 一张便签 = 一个独立的 WPF Window
**功能清单：**

- 桌面任意位置新建便签
- 纯富文本编辑（RichTextBox，支持 Ctrl+B / I / U）
- 拖四边四角改大小，带最小尺寸限制
- 左键拖便签顶部边缘区域移动窗口
- 删除 / 折叠 / 钉住（Topmost 置顶）
- 统一暗黄色 `#E8C547`
- 自动保存，启动恢复
- 开机自启
- 不透明，重叠由系统自然遮挡（后点的在上面）

**已砍掉：** Markdown、贴图

## 第 0 步：环境搭建（手动，一次性）

1. 装 **.NET 8 SDK**：dotnet.microsoft.com 下载 SDK（不是只装 Runtime）。装完 PowerShell 跑 `dotnet --version`，出现 8.x 即可。
2. 装 **VS Code**。
3. VS Code 装扩展：**C# Dev Kit**（微软官方，自动带 C# 扩展）。
4. 建项目：
   - 目录例如 `D:\projects\sticky-notes`
   - `dotnet new wpf -n StickyNotes`
   - `git init`
5. VS Code 打开该文件夹，按 F5 能弹空白窗口 = 环境通。
6. 在该目录起 Claude Code。

## 阶段一：单个便签窗口 — 显示 / 拖动 / 改大小

**指令：**

```
阶段一：先做一个 StickyNoteWindow（无边框 WindowStyle=None、AllowsTransparency=True、ResizeMode=NoResize 但我们自己实现缩放）。背景暗黄色 #E8C547，圆角，顶部留一条约 28px 高的"标题条"区域。要求：
1. 左键按住标题条区域可拖动整个窗口移动。
2. 自己实现四边和四角拖拽改变窗口大小，最小尺寸 150×120。
3. 标题条右侧放三个小按钮占位（钉住、折叠、删除），现在先只画出来，不用实现功能。
4. 程序启动时先弹出一个这样的便签用于测试。
先只做这一个窗口的交互，不要做多便签、不要做保存。完成后告诉我怎么运行测试。
```

**验收：** F5 弹出暗黄便签 → 拖标题条能移动 → 拖边角改大小、缩不到比 150×120 更小 → 三个按钮画出来了。全过才进阶段二。

## 阶段二：富文本编辑

**指令：**

```
阶段二：在便签标题条下方填满一个 RichTextBox，背景透明、无边框，字体颜色深色（在暗黄底上清晰）。支持 Ctrl+B 加粗、Ctrl+I 斜体、Ctrl+U 下划线（WPF RichTextBox 自带这些命令，确认绑定生效）。点便签正文区域能输入文字。注意：拖动移动只在标题条触发，不要影响正文区域的文字选择和编辑。
```

**验收：** 正文能打字 → Ctrl+B/I/U 生效 → 正文里选字、拖选不会误触发窗口移动。

**指令：**

```
阶段三：现在支持多张便签。
1. 用一个托盘图标（NotifyIcon）或一个全局快捷键来"新建便签"，新便签出现在屏幕中央偏移一点的位置，避免叠在一起。给我一个明确的新建入口。
2. 实现标题条上"删除"按钮：点了关闭并销毁这张便签窗口。
3. 多张便签各自独立，能分别移动、改大小、编辑，互不影响。重叠时由系统自然遮挡（后点的在上面）。
```

**验收：** 能建第二、三张 → 各自独立操作 → 删除能删单张 → 重叠时点击的浮到上面。

## 阶段四：钉住 + 折叠

**指令：**

```
阶段四：实现标题条两个按钮。
1. 钉住：点击切换该窗口 Topmost 真/假，钉住后这张便签浮在所有程序（包括别的软件）之上；按钮要有选中/未选中的视觉区别。
2. 折叠：点击后便签只保留标题条那一条（高度缩到只剩标题条），再点恢复原高度；折叠时记住原来的高度。
```

**验收：** 钉住后切浏览器仍浮最上 → 取消钉住会沉下去 → 折叠只剩一条 → 再点展开回原样。

## 阶段五：自动保存 + 启动恢复（最关键，重点验收）

**指令：**

```
阶段五：持久化。把所有便签的状态保存到 JSON 文件，存放在 %AppData%\StickyNotes\notes.json。每张便签要保存：富文本内容（用 XAML 字符串序列化 RichTextBox 内容，TextRange.Save 用 DataFormats.Xaml）、窗口位置 X/Y、宽高、是否钉住、是否折叠及折叠前高度。
触发保存：内容变化、移动、缩放、钉住、折叠、删除时都自动保存（做轻量防抖，避免频繁写盘）。
程序启动时读取该 JSON，恢复所有便签到原位置、大小、内容、状态。文件不存在就启动一张空白便签。
```

**验收（仔细测）：** 建几张、打字、调位置大小、钉一张、折一张 → 完全关掉 → 重启 → 所有便签连内容带格式带位置带状态原样回来。富文本格式恢复是最易出 bug 处，重点查。

## 预防针（重要）

1. **WPF 调试轮次比 Web 多**：XAML 错常在运行时才报。每次改完立刻 F5，有红色异常整段贴回去给 Claude Code，别攒着。
2. **AllowsTransparency=True + 无边框** 是 WPF 异形窗口标准姿势，但和某些显卡驱动/窗口阴影偶有摩擦。透明/圆角不对时让它查这个属性组合。
3. **富文本保存优先 XAML 序列化**（TextRange.Save + DataFormats.Xaml），还原最准。若用 RTF 出现格式丢失，换 XAML。
4. **移动 vs 改大小热区别重叠**：标题条管移动，四边四角管改大小。靠近角落分不清时把改大小触发边框设窄（如 6px）。
5. **暗黄色常量** `#E8C547`。想更亮接近便利贴用 `#F2D55C`；想更暗用 `#CDA434` / `#B8941F`。改一个常量即可。

## 内存 / 存储参考

- **常驻内存：** 约 30–60 MB（一便签一窗口，WPF 窗口很轻，多开十几张也只多几 MB）。
- **程序本体：** 自己用（依赖框架）1–5 MB；给别人（自包含裁剪）30–60 MB。
- **.NET 8 运行时：** 一次性系统共享，装 SDK 已含。
- **便签数据：** 每条 0.5–2 KB，上百条 < 1 MB。
